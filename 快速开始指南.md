## 🛠️ 下一步操作

### 第一步：安装前端依赖

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次运行需要几分钟）
npm install

# 启动开发服务器
npm run dev
```

访问 http://localhost:5173 即可看到项目

## 💡 开发建议

### 代码规范

- 使用 TypeScript 保证类型安全
- 遵循 ESLint 和 Prettier 代码规范
- 组件命名使用大驼峰（PascalCase）
- 函数命名使用小驼峰（camelCase）
- 常量使用全大写+下划线（CONSTANT_CASE）

### Git 提交规范

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建工具或辅助工具的变动
```

### 性能优化建议

- 使用路由懒加载
- 图片使用懒加载和 CDN
- 接口响应使用 Redis 缓存
- 使用虚拟滚动处理长列表
- 代码分割，按需加载

## 🔧 常见问题

### Q: 如何解决依赖安装失败？

A: 尝试删除 node_modules 和 package-lock.json，然后重新安装：

```bash
rm -rf node_modules package-lock.json
npm install
```

### Q: 如何连接数据库？

A: 修改后端的.env 文件，配置 DATABASE_URL：

```
DATABASE_URL=mysql://user:password@host:3306/database
```

### Q: 如何部署到 Vercel？

A:

1. 将代码推送到 GitHub
2. 在 Vercel 中导入项目
3. 配置环境变量
4. 自动部署
